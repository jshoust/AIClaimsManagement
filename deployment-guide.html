Boon AI Claims Processing - Deployment Guide

Important: This guide provides instructions for deploying the Boon AI Claims Processing application to your company's directory.

Prerequisites

  • Node.js (v16+) installed on your server
  • PostgreSQL database server
  • OpenAI API key for document analysis features

Required Files for Deployment

You need to upload the entire project directory to your server. The main directories and files include:

├── client/ # Frontend React application │ ├── public/ # Static files including Claim_Form.pdf │ ├── src/ # React source code │ └── index.html # Main HTML file ├── server/ # Backend Express server │ ├── services/ # Business logic services │ └── index.ts # Server entry point ├── shared/ # Shared code between client and server │ └── schema.ts # Database schema definitions ├── uploads/ # Directory for uploaded files ├── package.json # Project dependencies └── drizzle.config.ts # Database configuration

Deployment Steps

Step 1: Upload the Files

Upload all project files to your company's web directory, maintaining the folder structure.

Step 2: Install Dependencies

Run the following command in the project root directory:

npm install

Step 3: Set Up Environment Variables

Create a .env file in the project root with the following variables:

DATABASE_URL=postgres://username:password@hostname:port/database_name
OPENAI_API_KEY=your_openai_api_key

Step 4: Initialize the Database

Run the database migration and seed:

npm run db:push

Step 5: Build the Frontend

Build the React frontend:

npm run build

Step 6: Start the Application

Start the application server:

npm run start

The application should now be accessible at http://your-server-address:5000

Important Assets

These assets must be properly placed for full functionality:

  • Claim_Form.pdf - Ward Trucking claim form template, must be in the client/public directory
  • Boon logo and word.png - Brand logo used in the application header

Database Configuration

The application uses PostgreSQL with Drizzle ORM. The database schema includes tables for:

  • Users - User accounts and profiles
  • Claims - Claim details and status
  • Tasks - Follow-up tasks for claims
  • Activities - Tracking of claim activities
  • Documents - Uploaded files and analysis results
  • Email Templates - Templates for automated emails
  • State Laws - Reference data for state-specific laws

Additional Integration Notes

  • The app integrates with OpenAI for document analysis - ensure your API key has adequate permissions
  • Document storage is configured to save to the local filesystem in the uploads directory
  • Email functionality requires proper SMTP configuration if used in production
